## Pivots and decisions made mid-execution

1. **Go version raised from 1.22 to 1.25.** The plan baselined Go
   1.22 in §1's P0 cell. After Stream C landed `testcontainers-go`
   and its modules-minio submodule, transitive dependencies
   demanded Go 1.25. A downgrade was attempted (pinning older
   testcontainers releases) but reverted: every supported version
   pulled the same toolchain floor. We accepted Go 1.25 as the
   new v0.1 baseline. Consequences:
   - `go.mod` declares `go 1.25.0`.
   - `.github/workflows/ci.yml` matrix is `['1.25']` only (the
     plan's `['1.22','1.23']` matrix was tightened to a single
     point in the pivot).
   - `scripts/add-provider.sh` writes `go 1.25` into scaffolded
     provider modules.
   - ADR Follow-up #3 (`pkg/testkit/contract` module hoist) was
     promoted from "M6, conditional on velocity" to "v0.2.0
     mandatory" — see `RELEASING.md` §5 — so consumers who only
     want `pkg/uos` can drop the testcontainers cost.
2. **CI matrix scope.** The plan called for a 2×2 OS×Go matrix
   (`ubuntu-latest`/`macos-latest` × `1.22`/`1.23`). After the Go
   1.25 pivot, the Go axis collapsed to a single value (`'1.25'`)
   to keep CI cost flat; OS axis is preserved.
3. **Surface test expanded to three subtests.** Critic R1
   sign-off required pinning all three frozen sets at the same
   strictness; the original plan §7 mentioned 14 + 13 only. The
   final `TestFrozenSurface` covers all three and was
   mutation-verified before being shipped.

  (43 unit tests in pkg/uos + subpackages, plus 16 transfer

  subtests pass (`codes_frozen_14`, `capabilities_frozen_13`,
  `direct_grant_modes_frozen_4`).

  (The runtime `unit-docker` job exercises the actual MinIO

  verification.)
