## 1. Core Abstractions (frozen)

The following 11 entities form the v1 public surface. Drivers MAY NOT introduce new top-level entities visible from `pkg/uos`. Vendor-specific richness escapes via `As(target any) bool` only.

### 1.1 Entity inventory

| Entity | Defined in | Role |
| --- | --- | --- |
| `Provider` | `pkg/uos` | string identifier (e.g., `"aws"`, `"alibaba"`). |
| `Config` | `pkg/uos` | Top-level instantiation parameter; carries `DriverConfig any` for provider-specific options. |
| `Factory` / `Registry` | `pkg/uos` | SPI for provider factories; `Registry.Open(ctx, cfg)` resolves provider and returns `Client`. |
| `Client` | `pkg/uos` | Top-level handle. Composed of `BucketService`, `ObjectService`, `MultipartService`, `Signer`, `Capabilities()`, `As(target any) bool`, `Close()`. |
| `BucketService`, `ObjectService`, `MultipartService`, `Signer` | `pkg/uos` | Sub-service interfaces; subset use is allowed (testing, partial drivers). |
| `Capability`, `CapabilityStatus`, `CapabilityReport`, `Availability` | `pkg/uos/capability` | Per-(driver, config, credential) capability declaration; see ┬¦7.2. |
| `Error`, `Code` | `pkg/uos` | Single concrete `*Error` plus 14 frozen `Code` constants; see ┬¦7.1. |
| `Credential`, `Provider` (credential), `AuthScheme` | `pkg/uos/credential` | Pluggable credential chain. |
| `transfer.Manager`, `transfer.Config`, `transfer.StateStore` | `pkg/uos/transfer` | Multipart/resume orchestration. |
| `DirectGrant`, `SignedURL`, `SignURLRequest`, `DirectGrantRequest` | `pkg/uos` | Unified URL- and non-URL-based time-bounded grants (Azure SAS, Qiniu Token, Upyun FORM all map here). |
| `ObjectInfo`, `BucketInfo`, `ObjectReader`, `Checksum`, `Metadata`, `ContentHeaders`, request/response structs | `pkg/uos` | Request/response value types per PRD ┬¦4.4. |

### 1.2 Top-level interface signatures (binding)

```go
// pkg/uos
type Client interface {
    Provider() Provider
    Capabilities(ctx context.Context) (capability.Report, error)

    Buckets() BucketService
    Objects(bucket string) ObjectService
    Multipart(bucket string) MultipartService
    Signer(bucket string) Signer

    As(target any) bool
    Close() error
}

type Factory interface {
    Provider() Provider
    Validate(cfg Config) error
    Open(ctx context.Context, cfg Config) (Client, error)
}

type Registry interface {
    Register(f Factory) error
    Open(ctx context.Context, cfg Config) (Client, error)
}
```

Sub-service signatures (`BucketService`, `ObjectService`, `MultipartService`, `Signer`) follow PRD ┬¦4.3 verbatim. Where this freeze does not override a PRD field, take PRD as authoritative for that field's name and type.

1. Justification by Ōēź 2 independent providers OR by an unambiguous PRD section, AND

## 2. Non-Goals (explicit v1 exclusions)

The following are **out of scope**. Adding any of them is not a "small change" ŌĆö it requires re-running the freeze process.

### 2.1 Hard exclusions (repository scope)

- **Proxy server, gateway, SaaS control plane, credential-hosting service.** This repository builds a Go SDK / library only. (Hard rule from `AGENTS.md`.)
- Daemons, sidecars, sidecar protocols, or any process that mediates traffic.

### 2.2 Out-of-scope features (v1 SDK API surface)

- CDN domain management, accelerated-domain signing, CDN refresh / preheat.
- Image / audio / video processing, content moderation, transcoding, data pipelines.
- Bucket lifecycle, cross-region replication, object lock, legal hold, inventory, event notifications, website hosting.
- Cross-provider server-side copy. (Default helper = read+write streaming; never a `core` API guarantee.)
- A first-class `Directory` abstraction. Directories surface only via `List(prefix, delimiter)`.
- `Rename` / `Move` as core primitives. Default helper = `Copy` then `Delete`. Native rename, where supported, is exposed via `As(target)` extension only.
- Object key normalization (no `path.Clean`, no leading-slash trim, no UTF-8 transcoding).

### 2.3 Things the SDK guarantees it will NOT do

- Cross-provider consistency guarantees.
- Auto-promote ETag to MD5 / content hash. (`Checksum` is the only correct vehicle.)
- Disable TLS verification by default; doing so requires explicit opt-in and emits a warning at construction time.
- Log full Signed URLs, Authorization headers, SAS tokens, Upload Tokens, or other credentials.

## 3. Package Boundaries (multi-module Go layout)

### 3.1 Decision

The repository is **multi-module**: one root module hosts the public API and shared test helpers; each provider lives in its own `go.mod` and depends on the root module. This is the only layout that satisfies NFR-008 at the module level ŌĆö users only pay for SDK transitive deps of the providers they actually wire in.

### 3.3 Dependency direction (binding)

```
business code
    Ōåō
providers/<name>  ŌöĆŌöĆdepends-onŌåÆ  pkg/uos (+ pkg/uos/{capability,credential,transfer,middleware,httpx})
                                  Ōåæ
pkg/testkit/contract  ŌöĆŌöĆdepends-onŌöĆŌöĆŌöś   (importable by any provider's *_test.go)
```

Rules:

- `pkg/uos` MUST NOT import any `providers/<name>`.
- `pkg/uos` MUST NOT import any third-party cloud SDK.
- `pkg/testkit/contract` MAY depend on `pkg/uos` and on `testcontainers-go` / Docker SDK; nothing else.
- Each `providers/<name>` MAY import `pkg/uos` and exactly one vendor SDK (its own).
- No `providers/<a>` may import `providers/<b>`. Shared sub-routines for the S3 family go in `pkg/uos/httpx` or a future `pkg/uos/s3common` (deferred until at least 2 S3-family drivers have shipped).

### 3.4 `internal/` packages

Hard rule of multi-module: `internal/` packages are not importable across modules. Therefore:

- The PRD's `/internal/httpx` is **moved to `pkg/uos/httpx`** (public). Same applies to any future shared helper.
- A package may live under `internal/` only if it is consumed solely from within the root module (an implementation detail of `pkg/uos` itself).

### 3.5 Module path placeholder

The directory layout above uses `github.com/<owner>/object-storage-client` as the module path. The actual repository owner / host MUST be set on first `go mod init`. All cross-module imports use this placeholder until the owner is fixed.

### 3.6 `go.work` policy

`go.work` is committed. It enumerates the root module plus every `providers/<name>` module so that local `go test ./...` from the workspace root runs every provider without `replace` directives. Releases are still tagged per module; `go.work` does not affect downstream consumers.

## 6. Contract Test Strategy

### 6.1 Where the suite lives

`pkg/testkit/contract` exports a single entry point:

```go
func RunSuite(t *testing.T, factoryUnderTest FactoryUnderTest)
```

Each provider's `*_test.go` constructs a `FactoryUnderTest` (its own `factory.New(cfg)` + a `Setup(ctx)` returning a `Client` plus a fresh-bucket cleanup) and passes it in.

### 6.2 PR gate (mandatory)

- The suite runs against a **MinIO container spawned via `testcontainers-go`** (`pkg/testkit/contract/minio.go`).
- `providers/minio` runs the full suite; its tests gate the PR.
- `providers/aws` runs the suite by configuring the AWS SDK v2 to talk to the same MinIO endpoint (S3-compat mode); cases that strictly require real-AWS semantics are tagged and skipped in PR runs.
- Other providers (M3+) run the cases that are runnable against MinIO + their own validation hooks; cloud-only cases use `t.Skip` in PR runs.

Docker is therefore a hard build dependency for full PR test runs. Contributors without Docker can still build and run unit tests; Docker-gated tests will skip with a clear message.

### 6.3 Cloud nightly (opt-in)

- A separate workflow runs the **full** suite against real cloud endpoints.
- Triggered by per-provider repo secrets (e.g., `OMC_AWS_NIGHTLY_KEY`). If the secret is absent, the job for that provider exits SKIP, not FAIL.
- The same secret pattern works for forks and personal repos: a contributor configures their own creds for their own bucket and gets full nightly coverage; without secrets they still get the MinIO PR gate.
- Cloud nightly never blocks merges; it produces a status report and opens / updates a tracking issue when a regression appears.

### 6.4 What the contract suite verifies

Per the v1 frozen surface:

- **Bucket**: Create / Stat / List / Delete (`CapBucketCRUD`).
- **Object**: Put (small + multipart-promoted) / Get / Head / Delete / Exists / DeleteMany / Copy.
- **List**: prefix, delimiter, pagination via opaque continuation token.
- **Range read** (`CapRangeRead`).
- **Multipart**: Initiate / UploadPart / Complete / Abort, including resume from persisted state.
- **Signer**: SignURL round-trip (where applicable); DirectGrant shape.
- **Metadata**: round-trip of system + user metadata, lower-case key normalization, raw header preservation in `Extra`.
- **Error mapping**: at least one assertion per `Code` constant that the relevant case is reachable for that provider.
- **Capability gating**: for each capability listed `Unsupported` for a provider, the corresponding op MUST return `*Error{Code: ErrUnsupported, Capability: <cap>}`.

### 6.5 What the contract suite does NOT verify

- Real performance (covered in M6 benchmarks).
- Actual durability or cross-region behavior.
- Non-data-plane management ops (lifecycle, replication, etc.) ŌĆö these are explicit non-goals.

## 7. Error Model & Capability Model Boundaries

### 7.1 Error model ŌĆö frozen 14-code surface

`pkg/uos.Code` is a string-typed enum **frozen to exactly the following 14 values for v1**:

```
unsupported           invalid_argument      not_found             already_exists
permission_denied     unauthenticated       precondition_failed   conflict
rate_limited          timeout               temporary             checksum_mismatch
length_required       internal
```

Rules:

- Each provider's `error_map.go` MUST translate raw vendor errors into one of these 14 codes. Provider-specific richness goes into:
  - `Error.Message` (human-readable, vendor-flavored allowed),
  - `Error.SecondaryID` (vendor secondary identifier, e.g., AWS extended request id),
  - `Error.Cause` (chained original error; `errors.Unwrap` works through it),
  - `Error.Capability` (when `Code == ErrUnsupported`, MUST cite the capability that was missing).
- Business code can rely on `errors.Is(err, &uos.Error{Code: uos.ErrNotFound})`-style matching being valid forever in v1.
- Adding a new `Code` is a `pkg/uos` minor version bump, gated by Ōēź 2 providers needing the same new semantic.

Drivers MUST NOT register new `Code` values at runtime. Drivers MUST NOT panic on unmapped vendor errors; the catch-all is `ErrInternal` with a populated `Cause`.

### 7.2 Capability model ŌĆö frozen 13-capability surface + ExtensionOnly slot

`pkg/uos/capability.Capability` is **frozen to exactly the following 13 values for v1**:

```
CapBucketCRUD            CapObjectCRUD          CapListPrefixDelimiter
CapRangeRead             CapMultipartUpload     CapSignedURLRead
CapSignedURLWrite        CapDirectGrant         CapObjectTagging
CapVersioning            CapObjectACL           CapManagedEncryption
CapNativeMove
```

The `Availability` enum stays as PRD ┬¦4.5: `Unsupported / Conditional / Supported / ExtensionOnly`.

Rules:

- Each provider's `capabilities.go` MUST return a `capability.Report` that maps **every** of the 13 entries (no missing keys; missing ŌåÆ `Unsupported` with `Reason="not implemented"`).
- Provider-specific high-value abilities (e.g., AWS Object Lock, Azure Blob Snapshot, Qiniu Upload Token, Upyun media processing) are NOT added to this enum. Instead they:
  1. Surface via `As(target any) bool` to hand the caller the underlying SDK client, AND
  2. Are referenced from the matrix as "ExtensionOnly: see provider doc".
- A capability of value `ExtensionOnly` in a `Report` means: "the underlying provider can do it, but `pkg/uos` does not abstract it; reach for `As(target)`".
- Adding a 14th capability is a `pkg/uos/capability` minor version bump, gated by Ōēź 2 providers exposing the same semantic at the abstraction level.

### 7.3 Why frozen sets

The contract test suite (┬¦6.4) iterates the `Code` and `Capability` sets to verify driver completeness. If those sets were extensible at runtime, the suite would have nothing fixed to assert against and the abstraction's promise to business code (static `errors.Is`, static `if cap == ...`) would erode. Vendor-specific richness has clear escape hatches (`Cause` / `As(target)`); the frozen sets exist so business code can stay vendor-neutral.
