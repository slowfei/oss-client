US-001 + US-002 + US-003 done in one batch (their work overlaps and was verifiable in a single round-trip):

Hoist mechanics:
- `cd pkg/testkit/contract && go mod init github.com/maqian/object-storage-client/pkg/testkit/contract`
- `go work use ./pkg/testkit/contract` (from root)
- Wrote `pkg/testkit/contract/go.mod` with `go 1.25`, requires for `github.com/maqian/object-storage-client v0.0.0` + `testcontainers-go` v0.42.0 + `testcontainers-go/modules/minio` v0.42.0, plus a `replace github.com/maqian/object-storage-client => ../../..` directive (works alongside go.work; documented for removal at parent-tag time).
- `cd pkg/testkit/contract && go mod tidy` populated 50+ indirects (the testcontainers/Docker/moby/OTel chain) and bumped testkit's go directive to `1.25.0`.
- `go mod tidy` from root dropped EVERY require — root is now stdlib-only. `go.sum` is 0 lines.

Root reverts:
- Root `go.mod` go directive: `1.25.0` → `1.22` (matches PRD acceptance / original M1 plan).
- `go.work` go directive: `1.26.2` → `1.25.0` (must satisfy testkit's `1.25.0` minimum; `1.25` was misread by Go as < 1.25.0).
- `scripts/add-provider.sh`: `go 1.25` → `go 1.22` so future scaffolded providers track root.

Verification after hoist:
- root: `go build/vet/gofmt -l .` all silent; `go test -short -race -count=1 ./...` 5 packages PASS; `TestFrozenSurface` 3 subtests pass.
- testkit: `go build`, `go build -tags=docker`, `go vet`, `gofmt -l .` all silent.
- root `go.sum` line count: 0 (NFR-008 fully satisfied at module level for the first time).

Lessons:
- `go mod init` writes the LOCAL toolchain version into go.mod's go directive; explicit override needed when toolchain != intended floor.
- `go.work` go directive `1.25` is interpreted as < `1.25.0` by Go's version comparator; always use the patch-suffixed form when you want to match minimum.
- The `replace` directive in testkit is necessary for `go mod tidy` because tidy doesn't consult go.work for module resolution (only `go build`/`go test` honor go.work for package lookup).
- `go mod tidy` includes deps for ALL build configs (since 1.17), so testcontainers correctly landed in testkit's go.mod even though it's only imported under `//go:build docker`.
